# Lyric Video 16:9 v3 Visual Design

更新时间：2026-06-10

## 1. 标题与状态

- 标题：`lyric-video-16x9-v3` 燕云乐坊官方单曲 Visualizer 视觉方案
- 作者：Codex
- 状态：Visual design baseline, pending implementation plan
- 适用范围：真实 AI 出歌后的 16:9 MP4 成片、Claude Web v1 成品预览、公司社区发布素材交接
- 上游依据：PRD v0.3、技术方案 v0.2、`docs/specs/lyric-video-16x9-v2-product-design.md`、v2 成片质量复盘

## 2. v3 核心判断

v2 已经解决“工程可用”：Remotion 能读取音频和封面，MP4 有音轨，字幕在安全区，输出可被发布包引用。v3 不应继续堆播放器式组件，而要解决“作品好看”和“官方音乐作品感”。

v3 默认方向定义为：

> 燕云乐坊官方单曲 Visualizer。

它应该像一支官方 OST 单曲发布视频：一张高质量主视觉成为整支视频的灵魂，Remotion 负责让主视觉缓慢呼吸、让歌词稳定出现、让作品身份克制成立。画面不应像古风 PPT、游戏 HUD、K 歌模板、音乐播放器录屏或频谱素材站模板。

## 3. 方案比较

### 3.1 推荐方案 A：孤城月影 / Official OST Visualizer

默认落地方向。

- 以全屏 AI 主视觉为画面主体，不再使用明显“左封面卡片 + 右文字”的 UI 布局。
- 风格接近网易云音乐官方单曲视觉化、游戏 OST 视觉海报动态版、克制电影标题卡。
- 适合 2-4 分钟长时间观看，不疲劳、不廉价。
- 对工程要求可控：仍可用 Remotion、Image2 封面、句级歌词、轻可视化完成。

推荐原因：它最符合当前产品目标，既能保持燕云气质，也能把用户生成内容包装成“可发布作品”，并且不依赖复杂剧情视频或角色动画。

### 3.2 备选方案 B：乐坊档案 / Archive Identity

- 更偏品牌视觉系统，画面像音乐厂牌档案、黑白排版、编号、印记和作品元信息。
- 高级感强，但游戏世界氛围弱，普通用户可能觉得冷。
- 适合后续做片头、片尾、运营精选合集，不适合作为默认全曲模板。

### 3.3 备选方案 C：江湖行旅 / Cinematic Postcard

- 更像一支电影风景短片，强调雾、月、雪、远山、城门、人物背影和长镜头。
- 氛围上限高，但严重依赖封面/底板质量；如果图片不够好，整支视频会塌。
- 可作为后续高级模板或活动模板，不作为当前默认模板。

## 4. v3 视觉定位

关键词：

- 官方单曲
- 燕云世界
- 克制电影感
- 黑金与冷月色
- 水墨雾气
- 边塞、古城、江湖、夜雪、孤月
- 用户作品被正式包装

画面第一眼应该让人感觉：

1. 这是燕云相关音乐作品。
2. 这不是临时 Demo，也不是普通 AI 歌曲生成站。
3. 字幕清楚，能完整听歌。
4. 成片可以直接交给社区发布系统。

## 5. Image2 主视觉要求

v3 的成败很大程度取决于封面/底板质量。Image2 不再只生成“封面”，而是生成“官方单曲 Visualizer 主视觉底板”。

### 5.1 画幅与构图

- 默认生成 16:9，建议 1920x1080 或更高等比尺寸。
- 主体放在画面中上部或中部偏左/偏右，底部 22%-30% 预留歌词安全区。
- 不要生成任何文字、logo、水印、边框、唱片封面排版或海报标题。
- 画面中心要有可识别意象：孤城、关隘、月、雾、雪、灯、乐器轮廓、江湖人物剪影等。
- 避免主体过密，避免满屏复杂建筑细节。

### 5.2 推荐正向 Prompt 模板

实际实现时由 `CoverPromptAgent` 结合歌曲标题、摘要、歌词情绪和燕云引用填充：

```text
official game OST visualizer key art, inspired by ancient Chinese frontier and Jianghu atmosphere, a lonely moon above distant fortress gates, misty mountains, cold night air, subtle lantern light, restrained cinematic composition, premium music single cover background, spacious lower third for lyrics, no text, no logo, no watermark, high contrast ink wash mood, deep black, moonlit blue gray, muted dark gold accents, elegant, quiet, solemn, film still, 16:9
```

### 5.3 推荐负向 Prompt

```text
text, logo, watermark, subtitles, UI, player controls, poster typography, busy collage, anime cute style, cartoon, 3D plastic, neon cyberpunk, nightclub spectrum, overdecorated scroll, cloud border, cheap fantasy, low resolution, blurry face, malformed hands, duplicated people
```

### 5.4 兜底主视觉

如果真实 Image2 失败，默认平台封面不能继续使用当前工程感 SVG。v3 需要一个更像品牌资产的兜底底板：

- 纯深色背景。
- 远山/城门/孤月的极简剪影。
- 无文字。
- 为歌词保留下三分之一暗场。
- 可由 Image 生成后人工确认是否纳入仓库；未确认前只作为本地验证资产。

## 6. 画面结构

默认输出：1920x1080，30fps，H.264 + AAC。

### 6.1 背景主视觉层

- 使用 Image2 主视觉全屏铺底，`object-fit: cover`。
- 整首歌做极慢 Ken Burns：缩放范围 1.02-1.06，水平/垂直位移不超过 2%。
- 叠加暗角、细颗粒、轻雾层和局部月光/灯光，不做彩色光球、强渐变或大面积霓虹。
- 背景不再做重度 26px 以上全屏模糊；主视觉应被看见，而不是只作为卡片背板。

### 6.2 电影遮罩层

- 顶部与底部各有轻微暗场，帮助文字可读。
- 可以使用非常轻的 2.39:1 电影遮罩感，但不能明显压缩歌词安全区。
- 暗场是为了可读性，不是为了遮丑。

### 6.3 作品身份层

- 左上角或右上角显示小号品牌识别：
  - `燕云乐坊`
  - `YANYUN MUSIC HOUSE`
  - 作品编号短码
- 字体小、间距稳、透明度克制，不能抢歌名和歌词。
- 不显示供应商、object key、发布包、JSON、provider 等技术词。

### 6.4 标题层

- 片头展示歌名，1.8-2.5 秒内完成出现。
- 歌名可以居中或偏左，取决于主视觉主体位置。
- 字体建议偏正式：宋体/高对比衬线/现代黑体混合，不使用随手的系统大粗字。
- 主体播放阶段标题降级成小号信息，避免长期占据视觉中心。

### 6.5 歌词层

- 下三分之一安全区，默认句级歌词。
- 每屏 1-2 行，长句自动断行。
- 中文单行建议 16-20 字，极限不超过 22 字。
- 字号建议 58-70px，根据字数动态压缩到可读范围。
- 颜色：米白或近白，带极轻描边/阴影。
- 动效：淡入 8-12 帧，停留，淡出 8-12 帧；不弹跳、不旋转、不逐字乱飞。
- 当前句为主，上一句/下一句默认不展示，避免 K 歌 UI 感。

### 6.6 音频可视化层

- 可视化只做“音乐存在感”，不是主角。
- 默认放在歌词下方或底部进度线上方，宽度 45%-65%，高度 24-48px。
- 颜色用低透明米白、暗金或冷青灰。
- v3 首版可继续用确定性伪波形，但视觉上要更细、更轻、更像刻度或声纹。
- 后续 v3.1 再升级真实音频分析波形。
- v3.1 默认不得依赖未验证的 provider hard line-synced subtitles；Yunwu timestamped lyrics 当前为 `blocked_provider_path` 时，优先使用无硬同步字幕或弱歌词卡方案。

### 6.7 进度层

- 底部一条极细进度线，高度 2-3px。
- 不出现播放器按钮，不让用户误解为可交互控件。
- 进度线颜色可随歌曲情绪轻微变化，但不能变成彩色进度条。

## 7. 时间结构

### 7.1 片头：0-2.5 秒

- 黑场或极暗主视觉中，月光/灯光先出现。
- `燕云乐坊` 小标识和歌名淡入。
- 主视觉缓慢推近。
- 可有一次极轻金色印记闪现，但不做卷轴展开、印章乱飞。

### 7.2 主体：2.5 秒至倒数 3 秒

- 主视觉持续缓慢运动。
- 歌词按句级 timeline 稳定出现。
- 可视化和进度线低调运行。
- 副歌或情绪强段落只增强 5%-12% 的亮度/对比/雾气，不切换复杂场景。

### 7.3 片尾：最后 3 秒

- 歌词淡出。
- 回收歌名、燕云乐坊、玩家原创曲、作品编号。
- 最后一帧仍保留主视觉，不纯黑、不空白，适合当社区缩略图。

## 8. 字体与排版策略

首版不强依赖未授权字体。工程实现按以下降级链：

1. 若后续获得授权字体，标题可使用高质量中文宋体/标题衬线字体。
2. 当前实现先使用系统字体栈，但通过字重、字号、行高、透明度和布局改善观感。
3. 不使用手写毛笔字体作为默认歌词字体。
4. 不使用夸张描边、强阴影、发光字或斜体装饰字。

推荐排版比例：

- 品牌识别：24-30px。
- 歌名片头：72-96px，根据字数收缩。
- 歌词：58-70px。
- 作品编号/玩家原创曲：22-28px。

## 9. Remotion 工程落点

v3 继续使用 Remotion，不引入新的生产渲染引擎。

建议新增模板：

- `LyricVideo16x9V3`：默认候选模板。
- `VisualizerBackground`：主视觉、暗角、颗粒、雾气、缓慢推拉。
- `IdentityMark`：燕云乐坊识别和作品编号。
- `TitleIntro`：片头歌名。
- `LyricCinemaSubtitle`：下三分之一电影字幕。
- `SubtleAudioTrace`：低调声纹/波形。
- `EndSlate`：片尾收束。

v3 可以与 v2 并存，先通过 `RENDER_WORKER_COMPOSITION_ID=LyricVideo16x9V3` 显式启用。等样本质量通过后，再决定是否把默认模板从 v2 切到 v3。

## 10. 输入数据要求

沿用 v2 已经打通的输入能力：

- `audio_source_path` 或等价可读音频 URL。
- `cover_source_path` 或等价可读 Image2 主视觉 URL。
- `song_title`。
- `song_summary`。
- `lyrics` timeline。
- `duration_ms`。
- `template_id`。
- `safe_area`。

v3 建议在 metadata 中补充：

- `visual_style_id=official-ost-visualizer`
- `cover_prompt_profile=ost-key-art-16x9`
- `template_id=lyric-video-16x9-v3`

## 11. 降级策略

- Image2 主视觉失败：使用平台默认 v3 主视觉底板，不阻断成片。
- 歌词 timeline 缺失：显示歌名、摘要、声纹和进度，进入 no-lyrics 模式。
- 歌词过长：优先断行和缩字号，不允许溢出安全区。
- 音频不可读：不得生成无声 MP4，继续按视频渲染失败收口。
- 波形失败：降级为确定性静态声纹，不阻断成片。
- v3 渲染异常：可配置降级到 v2，但 metadata 必须记录降级原因。

## 12. 验收标准

| 项目 | 标准 |
|---|---|
| 第一眼 | 像官方单曲 Visualizer，不像工程样例、PPT、播放器或频谱模板 |
| 主视觉 | 全屏可见，主体清晰，底部歌词安全区足够暗且不拥挤 |
| 歌词 | 1-2 行稳定可读，不溢出，不遮挡主视觉核心 |
| 音频 | MP4 必须包含可解码 AAC 音轨，不允许无声假成功 |
| 动效 | 缓慢、克制、连续；不眩晕、不频闪、不廉价 |
| 可视化 | 存在但低调，辅助音乐氛围，不抢歌词 |
| 片头片尾 | 歌名、燕云乐坊、作品编号出现；最后一帧可作为缩略图 |
| 兼容性 | v2 与 v3 可并存，可通过配置切换 |
| 脱敏 | 不记录完整 Prompt、真实密钥、供应商原始 URL 或供应商 raw payload |
| 证据 | 至少输出一支本地样例和一支真实链路样例，通过 ffprobe 双轨检查和人工视觉复核 |

## 13. 非目标

- 本批不做 9:16 竖版。
- 本批不做逐字 K 歌高亮。
- 本批不做剧情镜头、角色动画或 AI 视频生成。
- 本批不接公司真实发布、分享、审核、账号、权益系统。
- 本批不替换 Remotion 渲染技术栈。
- 本批不提交未经确认的大体积图片、音频或视频资产。

## 14. 实施顺序建议

1. 生成或准备一张 v3 默认主视觉底板，用于本地样例验证。
2. 新增 `LyricVideo16x9V3` 与拆分组件，先不替换 v2 默认模板。
3. 完成片头、主体、片尾三段时间结构。
4. 优化歌词排版、暗场、安全区和长句断行。
5. 调整声纹/进度线，使其更轻、更像官方视觉元素。
6. 用 `render:sample` 输出本地样例，抽帧检查 0s、2s、主歌、副歌、片尾。
7. 再用公网真实完整链路输出一支真实样本，人工检查视觉质量和播放体验。
8. 按 `docs/specs/lyric-video-16x9-v3-1-quality-acceptance.md` 确认字幕策略和媒体质量。
9. 质量通过后，更新默认模板或保留为可选高级模板。
