## 1. 背景与问题

《燕云十六声》官方社区需要一种能够承接长期内容活动的 AI 作曲能力。社区内容分发形态以图文和视频为主，无法直接分发 MP3，因此平台不能止步于“生成音频”，而必须输出可以直接进入社区发布链路的视频素材包。

本产品要解决的问题包括：

1. 玩家有燕云故事、角色经历、剧情感受或二创想法，但缺少作词、作曲和视频包装能力。
2. 官方社区需要一个可复用的音乐内容生产底座，而不是每次活动临时开发工具。
3. AI 作曲模型可以生成歌曲，但需要燕云语料库、Prompt 编排、歌词确认、视频合成、发布包交接、失败兜底等产品化能力，才能达到正式商用标准。
4. 公司已有账号、审核、发布、分享、任务权益、社区互动等系统，本平台不应重复建设，而应预留清晰可插拔边界。
5. 本地阶段必须先跑通完整生成闭环，确保公司开发接入真实系统前已有可验证、可演示、可部署的商用级基础能力。

## 2. 产品定位

燕云 AI 作曲平台是面向《燕云十六声》内容生态的专属 AI 作曲平台。玩家可以通过移动端 H5/WebView 或 PC Web 提供自己的江湖想法、故事或歌词，平台基于燕云世界观、剧情文案、人物资料等语料库生成或优化歌词，再调用 MiniMax Music-2.6 生成完整歌曲，并通过 Image 2 生成专辑封面，最终合成为可在社区分发的 16:9 MP4 歌词视频发布包。

产品不是 Demo，不是一次性活动页，也不是面向任意主题的开放式 AI 作曲站。它应作为长期商用级平台能力存在，后续可以承接不同社区活动、创作者玩法、官方运营专题和内容传播需求。

本平台只负责创作、作词、出歌、封面、视频、预览和发布包交接。社区账号、审核、权益、发布、分享、点赞、评论、推荐流等能力由公司既有系统承接，本平台通过 Adapter 和发布包接口预留接入边界。

### 2.1 产品一句话定义

> 玩家讲述一段燕云江湖故事，平台将其写成歌词、谱成歌曲、生成封面，并输出可发布的视频作品。

### 2.2 产品形态

| 维度 | 定义 |
|---|---|
| 用户侧 | 移动端 H5/WebView + PC Web 的创作台、作品列表、成品预览 |
| 平台侧 | 作词、出歌、封面、视频、发布包生成 |
| 接入侧 | 账号、审核、发布、分享、任务权益、社区互动均通过 Adapter 或发布包接口接入 |
| 产物侧 | 内部保留音频、歌词、封面、时间轴、视频；对社区交付 MP4 发布包 |

## 3. 产品目标与非目标

### 3.1 产品目标

1. **完整作曲闭环**：从用户想法或歌词输入，到歌词确认、出歌、封面、视频合成、发布包交付，全链路可用。
2. **燕云内容质量**：歌词表达应符合燕云世界观、剧情氛围和文风要求，避免乱编硬设定。
3. **低门槛创作体验**：普通玩家不需要懂音乐术语，只需要输入想法、故事或歌词。
4. **可正式商用**：具备异步任务、失败兜底、可观测、成本控制、外部系统可插拔、上线开关等正式产品能力。
5. **可复用平台能力**：不绑定单一活动形态，后续可承接长期活动和不同运营包装。
6. **本地完整可跑通**：在接入公司真实系统前，本地必须用 Mock Adapter 跑通完整生成链路，形成可交付给公司开发的稳定基线。

### 3.2 非目标

| 非目标 | 说明 |
|---|---|
| 不做纯音乐入口 | 平台只支持带歌词和演唱的歌曲生成，不提供纯音乐生成入口 |
| 不做翻唱或参考音频 | 不开放上传参考音频、Cover、仿唱、复刻能力 |
| 不做现实艺人/歌曲模仿 | 不支持指定现实歌手、现实歌曲、厂牌、专辑、影视或游戏商业配乐进行模仿 |
| 不做平台内社交 | 点赞、收藏、评论、话题、社区推荐流由网易大神社区系统承接 |
| 不做平台内发布和分享系统 | 平台只生成发布包并记录交接状态，不生成社区标题、文案、话题标签，也不维护社区侧“已发布/下架/精选”等状态 |
| 不承诺精确歌曲秒级时长 | 歌曲实际时长由模型根据歌词、结构和风格生成，平台只提供篇幅控制 |
| 不支持多人协作 | 不做协作者、共同编辑、团队空间 |
| 不支持用户可见版本管理 | 不做同一作品多版本管理；重新生成视为新作品 |
| 不支持生成后编辑原作品 | 已生成作品不可覆盖编辑 |
| 不支持外挂字幕 | 字幕必须烧录进 MP4 |

## 4. 核心产品原则

1. **燕云限定，风格开放**：音乐风格不设白名单，玩家可以表达流行、摇滚、民谣、说唱、电子、戏腔等方向；但歌词主题、故事表达、世界观引用必须受燕云内容约束。
2. **先确认歌词，再进入高成本出歌**：MiniMax 出歌和视频合成属于高成本链路，必须在用户确认歌词和歌曲方案后再发起。
3. **视频优先**：平台主要交付物是 16:9 MP4 歌词视频，MP3 只作为内部资产留存，不作为社区分发形态。
4. **用户讲故事，系统处理音乐细节**：前台不暴露 BPM、调性、混音、Prompt、Negative Prompt 等专业字段。
5. **异步生成，不强迫等待**：歌词生成可前台等待，出歌、封面、视频合成必须异步执行。
6. **失败可恢复，成本可解释**：模型失败、视频失败、封面失败等需要有明确兜底策略和权益处理规则。
7. **外部系统可插拔**：账号、审核、发布、权益、社区互动均通过 Adapter 预留，当前本地阶段使用 Mock。
8. **运营可配置、可关闭、可降级**：Prompt、语料库、模型开关、模板、权益策略等必须可配置。
9. **生成过程可追溯**：每个作品需要记录模型版本、语料库版本、生成阶段、失败原因、trace_id 和发布包交接信息。
10. **商用级目标不降级**：本项目按正式商用平台建设，不为短期提速移除异步任务、对象存储、Provider/Adapter、语料检索、视频渲染、可观测等核心能力。

## 5. 用户角色

### 5.1 普通玩家

- 输入自己的燕云故事、角色经历、剧情感受或创作想法。
- 让平台帮自己写成歌词并生成歌曲。
- 获取可直接进入社区发布链路的视频作品。

### 5.2 深度创作者

- 自己写词，使用平台生成歌曲和视频。
- 对 AI 生成歌词进行有限续写、润色和燕云化表达。
- 通过更强的创作控制项表达个人风格。

### 5.3 社区运营

- 配置活动主题、推荐创作模板、Prompt 模板、封面模板、视频模板和语料库版本。
- 基于平台能力策划不同活动。
- 关注生成质量、生成成本、失败率、成品发布效果。

### 5.4 公司开发与接入方

- 接入网易大神社区账号系统。
- 接入审核系统。
- 接入发布系统。
- 接入分享系统。
- 接入任务权益系统。
- 接入社区点赞、收藏、评论等既有能力。

## 6. 核心用户旅程

### 6.1 旅程一：灵感成歌

```text
进入创作台
  ↓
选择“灵感成歌”
  ↓
输入燕云故事或想法
  ↓
可选填写情绪、场景、角色关系、歌曲风格、演唱倾向
  ↓
系统生成歌词草案和歌曲方案
  ↓
用户查看歌名、歌词、风格说明
  ↓
用户选择确认 / 润色 / 重写
  ↓
确认后进入出歌
  ↓
系统生成歌曲、封面、字幕时间轴、MP4
  ↓
用户进入成品预览页
  ↓
作品可预览，发布包已准备好，后续由社区发布系统接入
```

### 6.2 旅程二：填词成歌

```text
进入创作台
  ↓
选择“填词成歌”
  ↓
粘贴自写歌词
  ↓
系统检查长度、结构、燕云相关性和禁用表达
  ↓
系统补全结构标签并生成音乐 Prompt
  ↓
用户确认歌曲方案
  ↓
进入出歌、封面、视频合成
  ↓
生成 MP4 成品，并准备发布包供社区发布系统获取
```

### 6.3 旅程三：AI 写词 / 续写 / 润色

```text
进入歌词工坊
  ↓
选择完整写词 / 续写 / 润色
  ↓
输入想法或已有歌词
  ↓
系统结合燕云语料库生成或优化歌词
  ↓
用户在最多 2 次修改机会内调整歌词
  ↓
确认歌词
  ↓
进入出歌流程
```

### 6.4 旅程四：生成中与返回查看

```text
用户确认出歌
  ↓
作品进入生成中
  ↓
页面展示当前阶段：出歌 / 封面 / 视频合成
  ↓
用户可离开页面
  ↓
回到我的作品查看状态
  ↓
成功则进入成品页，失败则展示原因和可执行动作
```

| 生成中页 | 展示任务阶段、可离开、可返回查看 | 是 |

1. 移动端 H5/WebView 是首要体验对象，优先保证输入、确认、生成中、成品播放在手机屏幕上可用。

3. 移动端和 PC Web 使用同一套作品状态、发布包状态、失败原因和接口语义，不拆成两套产品逻辑。
4. 普通用户侧页面不使用“发布包”作为主按钮文案；“发布包”保留给接口、后台和公司接入方语境。

| 页面 | 移动端体验重点 | PC Web 体验重点 |
|---|---|---|
| 创作首页 | 主入口突出“讲一段江湖故事，生成歌曲”；示例可一键填入 | 可同时展示创作入口、示例和作品列表入口 |
| 灵感成歌页 | 首屏只保留故事输入框和主要按钮，高级选项默认收起 | 可展开更多创作控制项，便于批量调试 |

| 歌词工坊页 | 明确剩余 AI 改词次数，避免用户误以为可无限重写 | 展示改词历史摘要和当前版本 |
| 歌词确认页 | 明确确认后进入高成本生成、原作品不可再改、权益将被锁定 | 展示歌词、歌曲方案、风格摘要和风险提示 |

| 我的作品页 | 按状态快速筛选：待确认、生成中、已生成、失败 | 支持分页、状态筛选和调试入口 |

### 7.4 用户侧文案边界

用户侧文案应面向玩家理解：

- “作品已生成”优先于“发布包已生成”。
- “可进入后续发布流程”优先于“获取发布包”。
- “正在谱成歌曲 / 正在合成视频作品”优先于模型或 Worker 术语。
- “重新合成视频 / 复制为新作品 / 稍后再试”优先于只展示失败原因。

接入方、后台、接口文档中继续使用“发布包”“PACKAGE_READY”“PACKAGE_FETCHED”等技术和交接状态。

## 8. 创作入口需求

### 8.1 入口一：灵感成歌

灵感成歌是主入口，面向普通玩家。

#### 8.1.1 输入项

| 字段 | 类型 | 必填 | 说明 |
|---|---|---:|---|
| story_input | 多行文本 | 是 | 玩家想写成歌的燕云故事、想法或情绪 |
| mood | 枚举/标签 | 否 | 热血、悲壮、温柔、洒脱、诡谲、苍凉等；可扩展 |
| scene | 枚举/自由补充 | 否 | 清河、开封、荒野、雪夜、酒楼、战场等；可扩展 |
| relationship | 枚举/自由补充 | 否 | 重逢、离别、守诺、独行、群像、决战等 |
| music_style | 自由文本 | 否 | 玩家可自由填写音乐风格，不做白名单限制 |
| vocal_preference | 枚举 | 否 | 男声、女声、群像感、不指定；仅作为模型提示，不承诺强控制 |
| song_scope | 枚举 | 否 | 短篇、标准、长篇；不等同于精确时长 |

移动端默认只展示 `story_input` 和主按钮；其他字段作为高级选项或标签选择，不要求用户一次性填写。

#### 8.1.2 前台文案建议

主输入框标题：

> 你想把哪段燕云江湖故事写成歌？

示例：

> 我想写一首清河雨夜，少年侠客与故人重逢，最后各自远行的歌。

#### 8.1.3 系统行为

1. 对用户输入做基础检查。
2. 识别主题、情绪、场景、角色关系、风格倾向。
3. 检索燕云 Markdown 语料库。
4. 生成结构化创作 Brief。
5. 生成歌名、歌词、歌曲说明、MiniMax 音乐 Prompt、封面 Prompt 种子。
6. 输出到歌词确认页。

### 8.2 入口二：填词成歌

填词成歌面向已经有歌词的用户。

#### 8.2.1 输入项

| 字段 | 类型 | 必填 | 说明 |
|---|---|---:|---|
| lyrics_input | 多行文本 | 是 | 用户粘贴的完整或半完整歌词 |
| music_style | 自由文本 | 否 | 用户希望的音乐风格 |
| vocal_preference | 枚举 | 否 | 男声、女声、群像感、不指定 |
| song_scope | 枚举 | 否 | 短篇、标准、长篇 |

#### 8.2.2 歌词处理要求

1. 歌词长度必须控制在 MiniMax 可接受范围内。
2. 用户不需要手写结构标签；系统可自动补全 `[Verse]`、`[Chorus]`、`[Bridge]` 等结构。
3. 如果用户歌词与燕云完全无关，系统优先尝试燕云化转义；无法转义时提示用户修改。
4. 如果歌词缺少副歌或结构过散，系统可以给出结构整理建议。

### 8.3 入口三：AI 写词 / 续写 / 润色

该入口作为歌词工坊能力，支持用户在出歌前完善歌词。

| 能力 | 说明 |
|---|---|
| 完整写词 | 用户输入主题或故事，系统生成完整歌词 |
| 续写 | 用户输入几句歌词，系统续写为完整歌曲 |
| 润色 | 用户输入已有歌词，系统提升燕云感、押韵、可唱性和结构完整性 |
| 结构整理 | 自动补全歌曲段落标签 |

限制：

1. 每个草稿默认最多 2 次 AI 改词机会。
2. AI 改词机会包括润色、续写、重写中的任意动作。
3. 用户确认歌词后，进入出歌流程；出歌后不再编辑原作品。

## 9. 歌曲篇幅策略

MiniMax Music-2.6 的歌曲实际时长由模型根据歌词长度、段落结构和音乐 Prompt 生成。产品不承诺精确秒级时长，也不在前台提供“60 秒 / 120 秒 / 180 秒”这类强承诺选项。

前台提供“篇幅”选择：

| 篇幅 | 用户理解 | 内部处理方式 |
|---|---|---|
| 短篇 | 适合快速传播，歌词更精炼 | 结构更短，主歌 + 副歌为主 |
| 标准 | 适合完整歌曲，默认选项 | 主歌、副歌、第二段、结尾较完整 |
| 长篇 | 适合叙事表达，歌词更充分 | 增加 Bridge、Final Chorus 或更完整叙事段落 |

说明：

1. 实际音频时长以模型返回结果为准。
2. 平台需要记录实际时长，并据此生成视频时间轴和发布包 metadata。
3. 如果音频时长明显异常，进入质量检测与失败处理流程。

## 10. 歌词工坊需求

### 10.1 作词主链路

作词主链路使用 DeepSeek V4 Pro + 燕云 Markdown 语料库。

流程：

```text
用户输入
  ↓
意图识别
  ↓
检索相关燕云语料
  ↓
生成结构化创作 Brief
  ↓
生成歌词草案
  ↓
歌词质量评分
  ↓
输出歌词确认页
```

### 10.2 DeepSeek 结构化输出

DeepSeek 输出应采用结构化 JSON，至少包含：

```json
{
  "song_title": "夜渡清河",
  "song_summary": "一首关于雨夜重逢与江湖别离的武侠叙事歌曲",
  "lyrics": "[Verse 1]\n...",
  "sections": ["Verse 1", "Chorus", "Verse 2", "Bridge", "Final Chorus", "Outro"],
  "music_prompt": "Chinese wuxia pop ballad, cinematic, emotional, ...",
  "cover_prompt_seed": "rainy river town at night, lone swordsman, lanterns, ...",
  "yanyun_references": ["清河", "江湖", "渡口"],
  "risk_notes": [],
  "quality_score": 0.0
}
```

### 10.3 歌词质量标准

| 维度 | 要求 |
|---|---|
| 燕云相关性 | 主题、场景、情绪或叙事应与燕云世界有关 |
| 世界观准确性 | 不编造官方未确认的硬设定、人物关系或剧情结论 |
| 可唱性 | 每行长度适中，避免长散文和过密信息 |
| 结构完整性 | 至少包含主歌和副歌，标准篇幅应有完整歌曲结构 |
| 语言质量 | 避免堆砌古风词、硬押韵、空泛套话 |
| 情绪一致性 | 与用户输入的故事情绪保持一致 |
| 风格适配 | 能转化为 MiniMax 可理解的音乐 Prompt |

### 10.4 低质量歌词处理

1. 系统不应向用户展示明显不达标歌词。
2. 若质量评分低于阈值，系统可自动重写一次，不消耗用户的 2 次改词机会。
3. 自动重写仍不达标时，提示用户补充故事细节或调整输入。

### 11.1 语料库形态

### 11.2 语料库目标

1. 写得像燕云。

- 主要引用的设定标签。
- 生成时使用的 Prompt 模板版本。

## 12. 模型能力映射与产品决策

### 12.1 MiniMax Music-2.6

用途：最终歌曲生成。

平台使用能力：

- 使用 `prompt` 描述音乐风格、情绪、演唱倾向、编曲方向。
- 使用 `lyrics` 输入歌词。
- 使用 `audio_setting` 控制输出格式、采样率、码率等。
- 使用 `output_format=url` 或 `hex` 获取音频结果。
- 记录模型返回的实际生成时长、trace_id 和状态。

平台产品决策：

| 能力 | 产品决策 |
|---|---|
| 歌词 + prompt 生成歌曲 | 主能力 |
| lyrics_optimizer 自动生成歌词 | 不作为主链路，仅可作为兜底或实验开关 |
| 纯音乐生成 | 不开放 |
| Cover / 参考音频 | 不开放 |
| 精确指定时长 | 不承诺 |
| 输出音频 URL | 必须立即下载并转存，不能长期依赖临时 URL |

说明：MiniMax Music Generation API 支持使用 prompt 和 lyrics 生成歌曲；非纯音乐模式下 lyrics 有长度限制并支持结构标签；输出 URL 有过期限制，因此平台必须转存音频。

### 12.2 DeepSeek V4 Pro

用途：燕云作词中枢。

使用场景：

- 用户输入理解。
- 燕云语料整合。
- 歌名生成。
- 歌词生成、续写、润色。
- 结构化创作 Brief 输出。
- MiniMax 音乐 Prompt 生成。
- Image 2 封面 Prompt 种子生成。

### 12.3 Image 2

用途：专辑封面生成。

产品规则：

- 基于歌名、歌词摘要、歌曲情绪、燕云场景生成封面 Prompt。
- 通过预设 Prompt 模板约束画面，避免跑偏。
- 默认生成 1 张封面。
- 用户不满意时可重新生成 1 张。
- 如果 Image 2 生成失败，使用默认燕云主题封面兜底，不阻断 MP4 生成。

### 12.4 视频渲染服务

用途：将歌曲音频、封面、歌词时间轴合成为 MP4。

能力要求：

- 音频解码、转码、响度处理。
- 句级歌词字幕时间轴生成。
- 字幕烧录。
- 16:9 MP4 渲染。
- 渲染失败可单独重试，不重新出歌。

## 13. 出歌引擎需求

### 13.1 出歌前置条件

必须满足以下条件才能发起 MiniMax 出歌：

1. 用户已确认歌词。
2. 歌词长度在模型可接受范围内。
3. 歌词结构已规范化。
4. 音乐 Prompt 已生成。
5. 权益已锁定，或本地 Mock 已通过。
6. 内容边界检查已通过，或外部审核 Adapter 已返回可继续。

### 13.2 出歌任务流程

```text
创建出歌任务
  ↓
锁定权益
  ↓
调用 MiniMax Music-2.6
  ↓
获取音频 URL 或 hex
  ↓
下载/解析音频
  ↓
转存对象存储
  ↓
记录实际时长和模型 trace_id
  ↓
执行基础音频质检
  ↓
进入封面与视频合成
```

### 13.3 基础音频质检

| 质量项 | 最低要求 |
|---|---|
| 文件完整性 | 文件可正常解码和播放 |
| 实际时长 | 不低于平台配置的最低可接受时长 |
| 静音检测 | 不应出现异常长静音 |
| 峰值检测 | 不应出现明显爆音 |
| 结尾完整性 | 不应明显突然截断 |
| 语言异常 | 不应出现大量无关语言或异常重复 |
| 风格偏离 | 不应与用户风格描述严重冲突 |

说明：质检可以先以规则检测 + 抽样人工复核为主，后续可接入更强音频质量模型。

## 15. 视频工坊需求

### 15.1 默认视频规格

| 项目 | 默认规格 |
|---|---|
| 比例 | 16:9 |
| 分辨率 | 1920×1080 |
| 视频编码 | H.264 |
| 音频编码 | AAC |
| 帧率 | 30fps |
| 码率 | 8–12 Mbps，最终以社区发布系统要求为准 |
| 字幕 | 烧录进视频，不依赖外挂字幕 |
| 容器 | MP4 |

### 15.2 默认模板：歌词视频版

画面结构：

1. 开头 2–3 秒：歌名、作者昵称或 Mock 用户名、平台视觉包装。
2. 主体阶段：专辑封面、虚化背景、歌词字幕、频谱/波形或轻动效。
3. 结尾 1–2 秒：歌名回收、作品编号。

### 15.3 字幕策略

首期采用句级歌词字幕，不承诺逐字 K 歌式高亮。

原因：MiniMax 普通歌曲生成结果不一定返回逐句或逐字歌词时间轴。平台需要基于歌词结构、实际音频时长和段落权重生成近似句级时间轴。

字幕质量要求：

| 项目 | 要求 |
|---|---|
| 展示方式 | 每屏 1–2 行，长句自动断行 |
| 安全区 | 不贴边，不遮挡核心视觉 |
| 错位容忍 | 允许轻微错位，不允许整段歌词长期不显示或明显跳行错乱 |
| 可读性 | 字号、描边或阴影需保证在亮/暗背景下可读 |
| 兜底 | 字幕时间轴失败时切换为封面视频版 |

### 15.4 备用模板：封面视频版

触发条件：

- 歌词时间轴生成失败。
- 字幕渲染失败。
- 歌词内容无法稳定分段。
- 运营配置为不展示歌词。

画面包含：

- 专辑封面。
- 歌名。
- 频谱/波形。
- 轻动态背景。

### 15.5 视频验收标准

| 项目 | 验收标准 |
|---|---|
| 播放 | MP4 可在主流播放器和社区发布系统中播放，播放失败必须返回明确失败码 |
| 音画同步 | 视频时长与音频时长差异不超过配置阈值，首期建议不超过 1000ms |
| 字幕 | 字幕烧录成功，位于安全区内，移动端和 PC Web 均可读；长句应自动断行 |
| 封面 | 不拉伸、不变形，主体不被严重裁切；默认封面兜底也必须可用于视频首帧 |
| 音频 | 可正常解码，无异常长静音、明显爆音、编码失败 |
| 时长 | 视频时长应覆盖完整音频，不应提前截断 |
| 文件大小 | 不超过社区发布系统确认的最大文件大小；未确认前使用配置项控制 |
| 失败兜底 | 字幕失败、封面失败、渲染失败均有明确失败码、用户文案和可执行动作 |

## 16. 作品状态机

### 16.1 产品状态

平台内部保留以下状态：

| 状态 | 含义 | 用户可见 |
|---|---|---:|
| DRAFT | 草稿，用户已输入内容但未形成可确认歌词 | 是 |
| LYRICS_READY | 歌词和歌曲方案已生成，等待用户确认 | 是 |
| GENERATING | 用户已确认歌词，正在出歌、生成封面或合成视频 | 是 |
| GENERATED | MP4 发布包已生成 | 是 |
| FAILED | 生成失败，需展示原因和可执行动作 | 是 |

新增 `LYRICS_READY` 是为了避免在歌词未确认时直接进入高成本出歌链路。

### 16.2 生成阶段

`GENERATING` 下的内部阶段：

| 阶段 | 说明 |
|---|---|
| MUSIC_GENERATING | 调用 MiniMax 生成歌曲 |
| AUDIO_INGESTING | 下载、解析、转存音频 |
| AUDIO_CHECKING | 基础音频质检 |
| COVER_GENERATING | 调用 Image 2 生成封面 |
| TIMELINE_BUILDING | 生成句级歌词时间轴 |
| VIDEO_RENDERING | 合成 MP4 |
| PACKAGE_BUILDING | 生成发布包 |

### 16.3 状态流转

```text
DRAFT
  ↓ 生成歌词
LYRICS_READY
  ↓ 用户确认出歌
GENERATING
  ↓ 成功
GENERATED

DRAFT / LYRICS_READY / GENERATING
  ↓ 失败
FAILED
```

### 16.4 发布包交接状态

平台不维护社区发布状态，但需要维护发布包交接状态，便于公司开发接入排查。

| 状态 | 含义 |
|---|---|
| PACKAGE_NOT_READY | 发布包未生成 |
| PACKAGE_READY | 发布包已生成并通过本平台接入前检查，可被发布系统获取 |
| PACKAGE_FETCHED | 发布系统或接入方已获取发布包 |
| PACKAGE_EXPIRED | 发布包临时访问地址过期，需要重新签发 |
| PACKAGE_BLOCKED | 发布包生成后被审核或接入前检查阻断，不可对外获取 |

说明：`PACKAGE_FETCHED` 不代表社区已发布成功，只代表本平台已完成素材交接。

- 标记素材不可继续进入发布包。

本地阶段使用 Mock 审核通过。

### 19.3 QuotaAdapter：任务权益系统

平台不写死生成次数。

预留：

- 查询可用权益。
- 锁定权益。
- 成功后扣减权益。
- 失败后释放权益。
- 运营发放权益。
- 任务系统发放权益后同步。

### 19.4 PublishAdapter：社区发布系统

平台不做社区发布动作，但预留：

- 获取发布包。
- 推送发布包。
- 接收发布系统结果回调。
- 记录交接状态。

说明：当前产品不生成社区标题、文案、话题标签，这些由既有社区发布能力承接。

## 20. 权益与成本规则

### 20.1 权益原则

1. 权益能力由 QuotaAdapter 接入，平台本地阶段使用 Mock。
2. 平台不写死“每日次数”。
3. 权益应在正式出歌前锁定，在 MP4 发布包可用后扣减主要出歌权益。
4. 因模型失败、视频失败、封面失败、审核阻断导致作品不可用时，不应扣减主要出歌权益。
5. 失败重试、视频重渲染、发布包重新签发不应重复扣减主要出歌权益。

### 20.2 动作与权益建议

| 动作 | 权益处理建议 |
|---|---|
| 创建草稿 | 不扣权益 |
| 生成歌词草案 | 默认不扣主出歌权益，可按运营策略限制频次 |
| AI 改词 | 每个草稿默认 2 次，不扣主出歌权益 |
| 确认出歌 | 锁定权益 |
| 成功生成可用 MP4 发布包 | 扣减主出歌权益 |
| MiniMax 出歌失败 | 释放权益 |
| MP4 渲染失败 | 不扣主出歌权益，可单独重试或释放权益 |
| 封面重生 | 默认不扣主出歌权益，是否单独计次可配置 |
| 用户对歌曲不满意重新出歌 | 视为新生成，按权益策略处理 |

### 20.3 成本记录

每个作品需要记录：

- DeepSeek 调用成本。
- MiniMax 出歌成本。
- Image 2 封面成本。
- 视频渲染资源成本。
- 存储与 CDN 成本估算。
- 成功 / 失败 / 重试次数。

## 21. 异常与失败处理

| 异常场景 | 系统处理 | 用户展示 | 权益处理 |
|---|---|---|---|
| 用户输入过短或无效 | 提示补充故事细节 | “内容太少，补充一些人物、地点或情绪会更容易写成歌” | 不扣 |
| 用户输入偏离燕云 | 尝试燕云化转义，失败则提示修改 | “这个主题与燕云关联较弱，请补充燕云场景或故事” | 不扣 |
| 歌词质量低 | 自动重写一次 | 不展示低质量歌词 | 不扣 |
| 用户不满意歌词 | 允许 AI 改词，最多 2 次 | 展示剩余改词次数 | 不扣主出歌权益 |
| MiniMax 出歌超时 | 自动重试 1 次 | “歌曲生成较慢，系统正在重试” | 暂不扣，保持锁定 |
| MiniMax 出歌失败 | 标记失败，释放权益 | 展示失败原因和重新生成入口 | 释放权益 |
| 音频文件异常 | 标记失败或重试下载 | “音频生成异常，请重新生成” | 释放权益 |
| 封面生成失败 | 使用默认封面兜底 | “封面暂用默认样式，可稍后重试” | 不扣主出歌权益 |
| 视频渲染失败 | 单独重试视频渲染 | “视频合成失败，系统可重新合成” | 不扣主出歌权益，可按配置继续锁定或释放 |
| 发布包审核或接入前检查失败 | 标记发布包不可获取，保留排查信息 | “作品暂不可进入发布流程，请稍后重试或联系运营” | 不扣主出歌权益 |
| 发布包 URL 过期 | 重新签发 URL | 对用户无感，接入方可刷新 | 不扣 |

## 22. 生成耗时体验

完整作品生成包括：

```text
DeepSeek 写词/改词
  ↓
MiniMax 出歌
  ↓
音频下载与质检
  ↓
Image 2 封面生成
  ↓
歌词时间轴生成
  ↓
MP4 渲染
  ↓
发布包生成
```

体验规则：

1. 歌词生成阶段可前台等待。
2. 出歌、封面、视频合成阶段必须异步化。
3. 用户可离开生成中页面，稍后从我的作品进入。
4. 前台展示阶段进度，不承诺精确完成时间。
5. 如果任务超过配置阈值，展示“生成较慢”而不是无限转圈。
6. 如果任务最终失败，展示可理解原因和可执行动作。

建议阶段文案：

```text
正在整理你的江湖故事
正在写成歌词
正在谱成歌曲
正在生成专辑封面
正在合成视频作品
作品即将完成
```

## 25. 非功能要求

### 25.1 稳定性

- 生成任务必须异步化。
- Worker 重启不能丢任务。
- MiniMax 临时音频必须及时转存。
- MP4 渲染失败可以单独重试，不重新出歌。
- 外部 Provider 异常时应可熔断、降级或暂停入口。

### 25.2 可追溯

每个作品必须记录：

- 用户输入摘要。
- 歌词内容与结构化版本。
- 语料库版本。
- Prompt 模板版本。
- DeepSeek 请求 ID。
- MiniMax trace_id。
- Image 2 trace_id。
- 视频渲染任务 ID。
- 实际音频时长。
- 文件 checksum。
- 失败原因。
- 权益锁定、扣减、释放记录。
- 发布包交接状态。

### 25.3 安全与隐私

- 不在日志中明文输出 API Key。
- 不在普通日志中长期保存完整敏感输入。
- 用户输入、模型 Prompt、歌词内容按公司数据规范分级存储。
- 发布包 URL 应支持签名、过期和重新签发。
- Adapter 调用必须具备鉴权。
- 失败日志应保留足够排查信息，但避免不必要的个人信息暴露。

### 25.4 素材生命周期

| 素材 | 留存建议 |
|---|---|
| 草稿 | 可配置留存周期 |
| 歌词草案 | 随草稿保留 |
| MiniMax 临时 URL | 转存后不依赖，过期即不可用 |
| 内部音频 | 随作品保留，供视频重渲染和排查使用 |
| 封面图 | 随作品保留 |
| MP4 成品 | 随作品保留，供发布系统获取 |
| 失败任务中间文件 | 短期保留，便于排查 |
| 模型调用日志 | 按公司日志规范保留 |

## 27. 商用上线验收标准

### 27.1 功能验收

- 三个创作入口均可完成到 MP4 发布包。
- 歌词确认页可用。
- 生成中状态可查询。
- 成品页可播放 MP4。
- 发布包接口可返回标准字段。
- 外部 Adapter 可用 Mock 替换真实实现。
- 移动端 H5/WebView 和 PC Web 均可完成核心创作链路。

### 27.2 质量验收

- 歌词应符合燕云相关性和可唱性要求。
- 歌曲音频可播放，无明显解码、爆音、异常静音问题。
- 16:9 MP4 可播放，音画同步，无明显字幕遮挡或严重错位。
- 封面不严重跑偏，不出现明显与歌曲内容冲突的画面。
- 失败任务有明确原因码。
- MP4 文件大小、签名 URL 有效期、字幕安全区、音画同步阈值均可配置，并在验收时记录实际值。

### 27.3 稳定性验收

- 任务系统重启不丢任务。
- 供应商超时、失败、返回异常有兜底。
- 对象存储失败、视频渲染失败可追溯。
- 发布包 URL 过期后可重新签发。
- MiniMax 出歌失败默认自动重试 1 次；视频渲染失败可单独重试，不重新出歌。

### 27.4 接入验收

- AccountAdapter 可替换为网易大神账号系统实现。
- ModerationAdapter 可替换为公司审核系统实现。
- QuotaAdapter 可替换为社区任务权益系统实现。
- PublishAdapter 可替换为社区发布系统实现。
- 分享系统由公司既有能力承接，本平台只保证发布包、素材 URL、状态和元数据可交接。

## 30. 参考依据
